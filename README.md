# Game of Life (Консольная версия на C)

Проект представляет собой учебную практику на языке C с использованием терминального интерфейса и предназначен для демонстрации навыков разработки консольных приложений в портфолио.

## Описание

Программа реализует классическую "Игру в Жизнь" Джона Конвея с визуализацией в консоли. Для отображения состояния игрового поля используется библиотека `ncurses`, что позволяет наблюдать симуляцию в реальном времени.

Подробнее о Игре в Жизнь:

- [Сайт Игры в Жизнь](https://conwaylife.com/wiki/Main_Page)
- [Набор начальных состояний](https://conwaylife.appspot.com/library/)

## Начальные состояния

Начальные состояния задают стартовое расположение живых клеток и влияют на эволюцию симуляции. Они демонстрируют различные паттерны — от стабильных объектов до осцилляторов и летящих кораблей.

В папке `datasets` представлены текстовые файлы `1.txt`–`5.txt`:

- `1.txt` — **R-Pentomino** — хаотичное начало с долгим развитием, порождающее множество устойчивых объектов (methuselah).
- `2.txt` — **Gliders** — несколько независимых глайдеров, демонстрирующих периодическое движение по полю (spaceship).
- `3.txt` — **Dragon** — корабль (ship), движущийся на 1 клетку каждые 6 поколений.
- `4.txt` — **Blinker Fuse Diagonal 2** — диагональная конструкция из осцилляторов (blinker), разрушаемая специальным уничтожителем (eater).
- `5.txt` — **Electric Fence** — модуль с пушкой (gun) и уничтожителем (eater) для фитиля муравьёв (Ants).

### Создание собственного состояния

Чтобы создать своё начальное состояние:

1. Выберите опцию `0` (генерация пустого поля) в меню.
2. Файл с нулями будет создан как `0.txt` в папке `datasets/`.
3. Переименуйте файл в любое свободное имя (например, `6.txt`).
4. Отредактируйте содержимое, задав живые (1) и мёртвые (0) клетки.
5. После данных игрового поля можно указать любую дополнительную информацию — она не будет считываться при корректных размерах поля.

> [!WARNING] Все примеры рассчитаны на размер поля **25×80**. При изменении размеров поле должно соответствовать новым размерам.

## Подготовка среды Linux

Перед сборкой убедитесь в наличии:

1. Компилятора C (gcc или clang).
2. Библиотеки `ncurses`.

Для Windows рекомендуется использовать WSL и VS Code:

> [!NOTE] Запуск напрямую в Windows без WSL не поддерживается из-за зависимости от `ncurses`. Подробнее: [WSL](https://learn.microsoft.com/en-us/windows/wsl/install), [VS Code](https://code.visualstudio.com/docs/setup/windows), [WSL + VS Code tutorial](https://code.visualstudio.com/docs/remote/wsl-tutorial).

Дополнительно можно установить инструменты для проверки кода:

- `clang-format` — автоформатирование (конфигурация `.clang-format` в корне).
- `cppcheck` — статический анализ.
- `valgrind` — проверка утечек памяти.

Примеры установки (Ubuntu/Debian):

```bash
sudo apt update
sudo apt install build-essential libncurses5-dev clang-format cppcheck valgrind
```

Примеры использования проверок:

```bash
clang-format -i *.c
cppcheck --enable=all *.c
valgrind ./game_of_life
```

> [!TIP] Проверки не обязательны для сборки, но полезны при доработке кода.

## Возможности

- Выбор начальной конфигурации из `./datasets/`.
- Создание пустого игрового поля.
- Пошаговое обновление по правилам Conway's Game of Life.
- Визуализация состояния с цветовой индикацией возраста клеток.
- Управление скоростью через клавиши `A/Z`.
- Выход через `Q` или пробел.

## Сборка

Для компиляции используйте Makefile или gcc с `ncurses`.

> [!IMPORTANT] Makefile предполагает `ncurses` в стандартных путях; при нестандартном расположении настройте `LDFLAGS`.

### Сборка с Makefile

```bash
make       # сборка
make clean # удаление объектов и бинарника
make rebuild # полная пересборка
```

### Сборка вручную

```bash
gcc game_of_life.c -o game_of_life -lncurses
```

## Запуск

> [!CAUTION] Не запускайте в терминалах без поддержки управления экраном.

```bash
./game_of_life
```

Управление:

- Выбор начального состояния или создание пустого поля.
- Скорость: `A` — ускорить, `Z` — замедлить.
- Выход: `Q` или пробел.
